# Security Policy

## Supported Versions

Security fixes are handled on a best-effort basis for the latest code on `main` and the most recent maintained package version, when a package has been published.

Older versions may not receive security updates.

## Reporting a Vulnerability

Please do not report suspected vulnerabilities in public GitHub issues, pull requests, or discussions.

GitHub private vulnerability reporting is enabled for this repository and is the preferred way to submit a report securely.

If you cannot use GitHub private vulnerability reporting, use another private contact route already published by the maintainer. If no private route is available, open a minimal public issue requesting a secure contact method without including vulnerability details.

When you report a vulnerability, include the affected version or commit, the expected impact, reproduction details, and any suggested mitigations if you have them.

Reports will be reviewed as time allows. Please allow reasonable time for validation and coordinated disclosure before sharing details publicly.
